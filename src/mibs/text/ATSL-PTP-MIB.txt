    DESCRIPTION
        "The ATSL MIB module for PTP version 2, partly based on
        PTPBASE-MIB.

        This MIB defines management objects for PTP clocks and ports,
        which form PTP interfaces.

        PTP stands for  Precision Time Protocol, and is defined in
        IEEEE 1588 standard. It has different editions (2002, 2008,
        2019). This MIB is for 2008 version, the most extended.

        PTP offers a precise synchronization for clocks in systems
        that run packet networks. It may use Ethernet or UDP frames
        for transport and enables different systems where different
        clocks coexist and ideally synchronize to a unique time
        source that is called a grandmaster clock.

        Each of the running clocks may have several ports. Domain
        concept is used to group clocks into independent systems.
        PTP allows for sub-microsecond accuracy without demanding
        too much resources. Clocks implements dataset to store
        configuration and detected information.

        This MIB addresses IEEE-1588-2008 (PTPv2) standard when
        running default profile. Other possible profiles are telecom,
        utility and power but they are addressed by their own MIBs.

        Also, it is intended for ordinary and boundary clocks.

        Tables

        - 'ptpClockTable': It is used to list active PTP interfaces
        (clocks) in the device. Clock name and number of ports are
        displayed.

        - 'ptpClockXTable': Extends ptpClockTable and add
        configuration items like transport protocol (ETH/UDP) and
        profile.

        - 'ptpClockDefaultDS': List defaultDS per clock. DefaultDS
        have fields defined in IEEE-1588-2008 8.2.1.

        - 'ptpClockCurrentDS': List currentDS per clock. currentDS
        have fields defined in IEEE-1588-2008 8.2.2.

        - 'ptpClockParentDS': List parentDS per clock. parentDS
        have fields defined in IEEE-1588-2008 8.2.3

        - 'ptpClockTPDS': List timePropertiesDS per clock.
        timePropertiesDS have fields defined in IEEE-1588-2008 8.2.3

        - 'ptpClockPortDS': List timePropertiesDS per clock.
        timePropertiesDS have fields defined in IEEE-1588-2008 8.2.3

        ALBEDO Telecom devices that currently implement this MIB are:

        -Net.Time PTP/NTP Network Clock.

        References:

        [IEEE-1588-2008] IEEE Standard for A Precision Clock
            Synchronization Protocol for Networked Measurement and
            Control Systems, IEEE Std. 1588-2008, July 2008."

    SYNTAX          INTEGER  {
                     -- reserved(0), 0x00
                     -- reserved(1:5), 0x01 to 0x05
                        clockclass6(6),                          -- 0x06

                        clockclass7(7),                          -- 0x07
                        -- reserved(8), 0x08
                        -- reserved(9:10), 0x09 to 0x0A
                        -- reserved(11:12), 0x0B, 0x0C
                        clockclass13(13),                        -- 0x0D
                        clockclass14(14),                        -- 0x0E
                        -- reserved(15:51), 0x0F to 0x33
                        clockclass52(52),                        -- 0x34
                        -- reserved(53:57), 0x35 to 0x39
                        clockclass58(58),                        -- 0x3A
                        -- reserved(59:67), 0x3B to 0x43
                        -- otherprofiles(68:122), 0x44 to 0x7A
                        -- reserved(123:127), 0x7B to 0x7F
                        -- reserved(128:132), 0x80 to 0x84
                        clockclass248(248)                       -- 0xF8
                    }

PtpClockIdentity ::= TEXTUAL-CONVENTION
    DISPLAY-HINT    "1x:"
    STATUS          current
    DESCRIPTION
        "8-octet array that forms an identity that serves as a unique
         identifier for each PTP clock.
         Network byte order is assumed.

        See IEEE-1588-2008 Section 7.5.2.2.

        The clock identity is constructed from physical port MAC."
    SYNTAX          OCTET STRING (SIZE (8))

PtpClockTimeInterval ::= TEXTUAL-CONVENTION
    DISPLAY-HINT    "255a"
    STATUS          current
    DESCRIPTION
        "Corresponds to the TimeInterval struct indicated in
        IEEE-1588-2008 Section 5.3.2.

        It will be presented in the form of a character array.
        Network byte order is assumed."
    SYNTAX          OCTET STRING (SIZE (1..255))

ptpClockTable OBJECT-TYPE
    SYNTAX            SEQUENCE OF PtpClockEntry
    MAX-ACCESS        not-accessible
    STATUS            current
    DESCRIPTION
        "Table that contains information about PTP clocks in the device"
    ::= { ptp 1 }

ptpClockEntry OBJECT-TYPE
    SYNTAX          PtpClockEntry
    MAX-ACCESS      not-accessible
    STATUS          current
    DESCRIPTION
        "A table entry that contains information about a
        single ptp clock. There are as many entries as ptp clocks are
        in the device."
    INDEX { ptpClockIndex }
    ::= { ptpClockTable 1 }

PtpClockEntry ::= SEQUENCE {
        ptpClockIndex           TableIndex,
        ptpClockName            DisplayString
}

ptpClockIndex OBJECT-TYPE
    SYNTAX                     TableIndex
    MAX-ACCESS                 not-accessible
    STATUS                     current
    DESCRIPTION
        "A unique value, greater than zero, for each ptp clock.
         It is recommended that values are assigned contiguously
         starting from 1. The value for each block index must remain
         constant at least from one re-initialization of the entity's
         network management to the next re-initialization."
    ::= { ptpClockEntry 1 }

ptpClockName OBJECT-TYPE
    SYNTAX                     DisplayString
